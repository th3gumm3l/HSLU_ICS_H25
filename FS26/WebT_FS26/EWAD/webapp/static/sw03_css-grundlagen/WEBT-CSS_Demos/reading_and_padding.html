Changed reading/writing direction

Some languages use a different reading/writing direction. Such examples are Arabic, Hebrew, Chinese, Japanese, Korean or Mongolian. A more extensive list of languages using right-to-left script can be found under Languages using right-to-left scripts. To change these directions, there are a couple of different HTML and CSS settings that affect this behavior. Some of them are explained below.

Padding on the "left"

In the following example for Arabic script we can use the HTML attribute dir on the body or a specific element. See the W3C article about Structural markup and right-to-left text in HTML for more details about the dir attribute. In the below example, we set <td dir="rtl"> on the table cell containing the Arabic text to make it read right to left.

Now we can see that padding-left still inserts the space on the left of the text but padding-inline-start puts it on the right.

Changing direction even more

It is even possible to change to vertical reading as used in Chinese, Japanese, Korean or Mongolian. This can be done through the CSS property writing-mode (see also the MDN reference about the property for more details).

In the following examples, all paragraphs have writing-mode: vertical-lr set.

The property writing-mode also distinguishes between vertical and sideways modes. The difference between these two has to do with the rotation of certain characters and only becomes apparent when using Japanese/Chinese characters. An extensive discussion can be found in the W3C article about Styling vertical Chinese, Japanese, Korean and Mongolian text.
